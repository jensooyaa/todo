# 认证流程

## 注册

```
用户输入密码 "123456"
        ↓
bcrypt.hash("123456", 10) → "$2a$10$xK3f..."
        ↓
存入数据库（只存哈希值，原始密码丢弃）
```

## 登录

```
用户输入密码 "123456"
        ↓
数据库查出哈希值 "$2a$10$xK3f..."
        ↓
bcrypt.compare("123456", "$2a$10$xK3f...") → true，验证通过
        ↓
jwt.sign({ userId, username }, JWT_SECRET) → 生成 token
        ↓
res.cookie('JWT', token, { httpOnly: true }) → 写入浏览器 cookie
        ↓
返回 { message, userId, username }（不返回 token）
```

## 后续每次请求

```
浏览器自动带上 cookie（withCredentials: true）
        ↓
auth 中间件：req.cookies.JWT → 取出 token
        ↓
jwt.verify(token, JWT_SECRET) → 解码出 { userId, username }
        ↓
挂到 req.user → controller 里可以用 req.user.userId
        ↓
验证失败或过期 → 返回 401 → 前端拦截器跳转登录页
```

## 安全分工

| 环节 | 方案 | 防什么 |
|------|------|--------|
| **存储** | bcrypt 哈希 | 数据库泄露后密码不可逆 |
| **传输** | HTTPS（部署时） | 中间人窃听 |
| **身份验证** | JWT + httpOnly cookie | 伪造身份 |
| **防 XSS 窃取 token** | httpOnly: true | JS 无法读取 cookie |
| **token 过期** | expiresIn: 1h | 限制 token 有效期 |
